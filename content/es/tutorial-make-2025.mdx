<h2>2) Conceptos básicos que importan</h2>
<h3>Visual scenarios &amp; mapping</h3>
<p>
          Construir automatizaciones como <em>escenarios</em> sobre un lienzo. Las entradas y salidas de cada módulo se convierten en campos en el
          <strong>Cartografía</strong> panel, donde arrastrar datos, transformar valores (funciones), y navegar arrays/collections. Usted puede ejecutar un solo módulo para “aprender” su estructura de salida, luego mapear con confianza a los próximos pasos.
        </p>
<h3>Routers, iterators &amp; aggregators</h3>
<p>
<strong>Routers</strong> ramificar su flujo en múltiples rutas con filtros y una posible <em>retroceso</em> ruta (procesada secuencialmente).
          <strong>Alfabetizadores</strong> dividir un array en un paquete por elemento; utilizar un <em>array aggregator</em> después, si necesita volver a registrar los resultados.
        </p>
<h3>Triggers: votación vs webhooks</h3>
<p>
          Los módulos de votación verifican los cambios en el horario. <strong>Webhooks</strong> recibir solicitudes HTTP y normalmente desencadenar escenarios al instante (paralela por defecto). Si prefiere el procesamiento ordenado o tiene recursos compartidos (por ejemplo, una API limitada por tarifas), active <em>secuencial</em> ejecución.
        </p>
<h3>Manejo de errores &amp; ejecuciones incompletas</h3>
<p>
          Adjuntar controladores de error (ignore, curriculum vitae, commit, rollback, break) para manejar fallos esperados sin desactivar su escenario. <em>ejecuciones incompletas</em> para almacenar el estado de una carrera fallida para posterior reingresar y evitar la pérdida de datos. Para problemas transitorios (limites de tasa, plazos), Hacer aplica <em>retroceso exponencial</em> para repeticiones automatizadas.
        </p>
<h3>Data Stores &amp; Data Structures</h3>
<p>
<strong>Almacenes de datos</strong> son la base de datos integrada y ligera para los datos clave del negocio (conjuntos de asistencia, cursores, caches id).
          <strong>Estructuras de datos</strong> describir formalmente los esquemas JSON/XML/CSV por lo que la asignación es predecible y validada.
        </p>
<h3>Agentes AI &amp; context</h3>
<p>
          Agents can use your scenarios as tools, keep <em>context</em> (archivos/texto) como memoria a largo plazo, y ejecutar dentro de los límites de tiempo/paso que configura. Puede utilizar el proveedor de inteligencia artificial de Make (uso de crédito dinamico) o su propia conexión AI en los planes Pro+.
        </p>

<h3>B) Construir una ingesta de plomo en tiempo real (Webhook - título CRM)</h3>
<ol>
<li><strong>Trigger:</strong> Webhooks • <em>Webhook personalizado</em>. Copiar la URL; su página de formulario/instalación será POST aquí.</li>
<li><strong>Validación:</strong> Añadir un <em>JSON √≥ Parse JSON</em> con una <em>Estructura de datos</em> (definir los campos requeridos). Si es inválido, ruta a un controlador de error que alerta Slack.</li>
<li><strong>Routing:</strong> Añadir un <em>Router</em> para VIP vs guías estándar (filtros en UTM/source o gastar). Marcar un <em>ruta de caída</em>.</li>
<li><strong>CRM upsert:</strong> Utilice el módulo CRM nativo (o HTTP) para <em>encontrar- contactos/actualización</em>. Considerar <em>secuencial</em> escenario si la API de CRM impone límites de tarifas estrictos.</li>
<li><strong>Notificar:</strong> Slack o resumen de correo electrónico (campos reducidos). Incluye un enlace de vuelta a tu registro de CRM.</li>
<li><strong>Observabilidad:</strong> Habilitación <em>ejecuciones incompletas</em> y crear un peso ligero <em>Data Store</em> para llaves de dedupe (email+source).</li>
</ol>

<div className="stack">
<h2>5) Confiabilidad: errores, registros, límites de tarifas</h2>
<h3>Webhooks &amp; throughput</h3>
<p>
          Webhooks instantáneo ejecutan inmediatamente y se ejecutan <em>paralelo</em> por defecto. Si los asuntos de orden estricto (pagos, acciones), cambiar <strong>procesamiento secuencial</strong>. Hacer puede manejar hasta <strong>30 solicitudes/segundo webhook</strong> por webhook; arriba que obtendrá HTTP 429.
        </p>
<h3>Desplazamiento exponencial predeterminado</h3>
<p>
          Para errores transitorios (<code>RateLimitError</code>, <code>ConnectionError</code>, <code>ModuleTimeoutError</code>), Hacer calendarios automáticos re-runs with increasing delays. Con ejecuciones incompletas habilitadas, la escalera de retry va ~1 min - título 10 min - título ... hasta horas antes de desactivar el horario si los fallos persisten.
        </p>
<h3>Manejadores de error (5 tipos)</h3>
<ul>
<li><strong>Ignoro</strong> (drop/continue), <strong>Resumen</strong> (continúe con los defectos), <strong>Commit</strong> / <strong>Rollback</strong> (para módulos compatibles con ACID), <strong>Romper</strong> (Ejecución incompleta de fuerza).</li>
</ul>
<h3>Patrón práctico</h3>
<ol>
<li>Habilitación <em>ejecuciones incompletas</em> y <em>secuencial</em> al escribir al mismo registro para evitar las condiciones de raza.</li>
<li>Utilice una entrada de Data Store para guardar idempotency (por ejemplo, hash de payload).</li>
<li>Ponte. <em>Respuesta de Webhook</em> cerca del final, si confías en el comportamiento de “el escenario de inhabilitación del terror” para la visibilidad.</li>
</ol>
</div>
</section>

<h2>6) Data Stores, Data Structures &amp; mapping</h2>
<h3>Almacenes de datos</h3>
<p>
          Piense en las tiendas de datos como tablas rápidas y estructuradas dentro Haga: perfecto para dedupe, cursores incrementales, pequeños diccionarios de referencia, o colas temporales. No son un reemplazo para su DB principal, pero eliminan un montón de código de pegamento para pequeñas necesidades de estado.
        </p>
<h3>Estructuras de datos</h3>
<p>
          Defina y reutiliza los esquemas para JSON/XML/CSV. Las estructuras validan los insumos/salidas y hacen predecible el mapeo. Utilice el generador al pegar una carga útil de muestra, luego los tipos de pinzas y los campos requeridos.
        </p>
<h3>Mapping arrays</h3>
<p>
          Puede indexar elementos de array en la interfaz de usuario de mapeo y combinar funciones de array para seleccionar valores por clave. Cuando no tiene información de estructura (por ejemplo, después de Parse JSON), ejecute el módulo una vez por lo que haga “aprenda” su salida y expone campos abajo.
        </p>
